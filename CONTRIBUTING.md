# Contributing to Gofasta CLI

Thank you for your interest in contributing to the Gofasta CLI! This document explains how to get started.

## Ways to Contribute

- **Report bugs** — Open an issue describing what happened, what you expected, and steps to reproduce.
- **Suggest features** — Open an issue describing the use case and proposed solution.
- **Submit code** — Fork the repo, make changes, and open a pull request.
- **Improve docs** — Fix typos, clarify explanations, or add examples.
- **Add skeleton templates** — Improve the project files generated by `gofasta new`.
- **Add generation templates** — Improve or add new code generators.

## Development Setup

1. **Fork and clone** the repository:

   ```bash
   git clone https://github.com/YOUR_USERNAME/cli.git
   cd cli
   ```

2. **Verify Go version** (1.25.0 or later required):

   ```bash
   go version
   ```

3. **Install dependencies:**

   ```bash
   go mod tidy
   ```

4. **Install golangci-lint** (optional, but recommended):

   ```bash
   go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest
   ```

5. **Build the CLI:**

   ```bash
   go build -o /tmp/gofasta ./cmd/gofasta/
   ```

6. **Test it:**

   ```bash
   /tmp/gofasta --help
   /tmp/gofasta new /tmp/testproject
   ```

## Project Structure

```
cli/
├── cmd/gofasta/main.go           # Entry point
├── internal/
│   ├── commands/                  # CLI commands (new, dev, migrate, etc.)
│   ├── generate/                  # Code generation engine
│   │   ├── templates/            # Go template strings for generated code
│   │   └── ...
│   └── skeleton/                  # Embedded project templates
│       └── project/              # Files that become a new project
├── dist/                          # Distribution files (shell-script installer)
└── go.mod
```

### Where to make changes

| If you want to... | Edit files in... |
|---|---|
| Add a new CLI command | `internal/commands/` |
| Change what `gofasta new` generates | `internal/skeleton/project/` |
| Change what `gofasta g scaffold` generates | `internal/generate/templates/` |
| Fix how files are patched after generation | `internal/generate/patcher.go` |
| Add a new field type (e.g., `json`) | `internal/generate/fieldparse.go` |
| Change project config defaults | `internal/skeleton/project/config.yaml.tmpl` |

### Template files

Files in `internal/skeleton/project/` ending in `.tmpl` are processed by Go's `text/template` during `gofasta new`. They use these variables:

- `{{.ProjectName}}` — The project name as provided (e.g., `myapp`)
- `{{.ProjectNameLower}}` — Lowercase (e.g., `myapp`)
- `{{.ProjectNameUpper}}` — Uppercase (e.g., `MYAPP`)
- `{{.ModulePath}}` — Go module path (e.g., `github.com/myorg/myapp`)

Files **without** `.tmpl` are copied as-is.

## Making Changes

1. **Create a branch** from `main`:

   ```bash
   git checkout -b your-feature-name
   ```

2. **Make your changes.**

3. **Build and test** the full flow:

   ```bash
   # Build the CLI
   go build -o /tmp/gofasta ./cmd/gofasta/

   # Generate a test project
   rm -rf /tmp/testproject
   /tmp/gofasta new /tmp/testproject

   # Verify the generated project compiles
   cd /tmp/testproject
   go build ./...
   ```

4. **Run unit tests:**

   ```bash
   go test ./...
   ```

5. **Commit** with a clear message:

   ```
   commands/new: add --no-git flag to skip git init

   Some CI environments don't have git available. This flag lets users
   skip the git init step during project creation.
   ```

6. **Push and open a pull request** against `main`.

## Pull Request Guidelines

- Keep PRs focused — one logical change per PR.
- If you change skeleton templates, verify the generated project compiles.
- If you change generation templates, test with `gofasta g scaffold`.
- If your PR addresses an issue, reference it: `Fixes #123`.

## Code Style

- Run `golangci-lint run` before submitting a PR. The repository's `.golangci.yml` configures all required linters.
- Follow standard Go conventions (`gofmt`, `go vet`).
- Keep the CLI's dependencies minimal — it should not import the `github.com/gofastadev/gofasta` library.
- Template files should produce clean, readable Go code.

## Reporting Security Issues

If you discover a security vulnerability, **do not open a public issue.** See [SECURITY.md](SECURITY.md) for instructions.
